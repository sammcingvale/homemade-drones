# v1 Field Kit

What to bring to a field flight session with the v1 airframe.

## Aircraft + power

- [ ] v1 airframe (1×)
- [ ] LiPo packs, charged: 4× (cat-008)
- [ ] Spare LiPo: +1× if available
- [ ] LiPo bag (transport)
- [ ] Battery straps: 2× spare

## Spare parts (for in-field repair)

- [ ] Spare props: 8× individual props (cat-004) — 2 full sets
- [ ] Spare motor: 1× (cat-003)
- [ ] Spare M3 SHCS, 12mm + 16mm: ~10 each (cat-011)
- [ ] Spare M3 nyloc nuts: ~10 (cat-012)

## Tools (mechanical)

- [ ] 2.5mm hex driver (Wera)
- [ ] M5 nut driver / rubber prop wrench
- [ ] Side cutters
- [ ] Tweezers
- [ ] Loctite 243 (small bottle for motor screw replacement)

## Tools (electrical / diagnostic)

- [ ] Multimeter
- [ ] USB cable (USB-C, for FC reflash if needed)
- [ ] Smoke stopper (XT60 inline) — for any post-crash power-up

## Ground station

- [ ] Toughbook (charged)
- [ ] SiK ground radio (915MHz USB-A)
- [ ] USB-A extension cable
- [ ] RadioMaster Pocket transmitter (charged)
- [ ] USB-C cable for TX charging (optional, for long sessions)

## Safety + admin

- [ ] FAA registration card (or printout)
- [ ] TRUST certificate (printout or phone screenshot)
- [ ] Phone with B4UFLY app
- [ ] Sand bucket or fire extinguisher (Class D)
- [ ] First aid kit (small)
- [ ] Eye protection (cheap safety glasses)
- [ ] Sunshade for laptop (folding cardboard works fine)

## Logging

- [ ] Notebook + pen for in-field observations
- [ ] Phone for build/flight photos

---

## Pack-out workflow

Before leaving the workshop:

1. Charge all LiPos to **flight charge** (4.20V/cell). Confirm cell balance ≤0.02V deviation per pack.
2. Run the airframe through the safety checklist (`docs/safety-checklist.md` → Pre-flight).
3. Check the weather. <8 m/s sustained wind is the hard ceiling.
4. Verify flight area on B4UFLY.
5. Pack this kit. Tick boxes physically.

## Pack-in workflow

After returning from the field:

1. Set all LiPos to storage charge (3.80V/cell) within 24 hours.
2. Visual inspection of airframe — log any damage in build's `flight-logs/<latest>/notes.md`.
3. Note anything used or damaged in the field for follow-up restocking.
4. Note any tools left at the field site (it happens).
